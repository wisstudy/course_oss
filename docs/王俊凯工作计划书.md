#### API-01: 课程级一键计算 (Tier 1 + Tier 2)

```
POST /api/teacher/calculate?course_id={course_id}&class_id={class_id}
Authorization: Bearer <token>
```

**核心逻辑**：
- **Tier 1 — 课程目标级 C̄j**：Pandas DataFrame 分组聚合 —— `GROUP BY student_id, objective_id` → `Cij = Σscore / Σmax_score` → `C̄j = mean(Cij)`
- **Tier 2 — 课程级 Ek**：查 `internal_weight_matrix` → `Ek = Σ(C̄j × w)`
- **锁机制**：结果写入 `calculation_results`（`locked=true`），禁止后续成绩导入
- **返回**：`{ objective_results: [...], course_results: {...} }`

#### API-02: 计算进度看板

```
GET /api/academic/calculation/progress
Authorization: Bearer <token>
```

**核心逻辑**：联表查询全部课程班级，关联 `calculation_results` 检查 `level='course' AND locked=true` 状态

#### API-03: 专业级全局计算 (Tier 3)

```
POST /api/academic/calculate/program
Authorization: Bearer <token>
```

**核心逻辑**：
- 前置校验：检查每个指标点的全部支撑课程是否均已锁定
- `Gk = Σ(Ek × W)`（聚合所有支撑课程的 Ek 与宏观权重 W 加权求和）
- 事务内执行（REPEATABLE READ）

#### API-04: 成绩解锁

```
POST /api/academic/unlock
Authorization: Bearer <token>
Body: { course_id, class_id, reason }
```

**核心逻辑**：`calculation_results.locked` → `false`；`INSERT INTO unlock_log`

#### API-05: 专业级报告数据

```
GET /api/academic/results/program
Authorization: Bearer <token>
```

**核心逻辑**：查全部 `level='program'` 的结果，返回 `[{ indicator_id, achievement_value }, ...]`

#### API-06: 专业级 Excel 报告导出

```
GET /api/academic/reports/program-excel
Authorization: Bearer <token>
```

**核心逻辑**：生成多 Sheet Excel：
- Sheet 1 "专业达成度汇总"（指标点 | Gk | 支撑课程数）
- Sheet 2+ 每指标点详情（课程 | 班级 | Ek | W | 贡献值 Ek×W）

## 七、风险与注意事项

1. **数据库模型协调**：计算引擎依赖 `calculation_results`, `macro_support_matrix`, `internal_weight_matrix`, `student_scores`, `unlock_log` 等表，需与钟崇涛/邓江鹏确认模型字段及表名的一致性
2. **浮点精度**：达成度计算采用 `DECIMAL(10,6)` 精度，权重校验容差 `±0.001`
3. **事务安全**：Tier 3 全局计算前置校验（所有支撑课程均已锁定）与计算逻辑必须在同一事务内完成
4. **ECharts 雷达图**：指标点数量动态变化，雷达图轴数需动态渲染
5. **并发控制**：同一课程的计算请求需防止重复提交（前端按钮 loading + 后端检查 locked 状态）
