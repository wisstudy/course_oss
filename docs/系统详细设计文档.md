## 2. 核心模块设计

### 2.1 认证授权模块 (Auth Service)

**对应子模块**：M0-1 登录认证、M0-3 用户管理

**技术**：FastAPI Security + OAuth2 (JWT) + bcrypt

**功能**：
- 用户登录认证，签发 JWT Token（有效期 8 小时）
- Token 身份标识与角色权限获取
- 用户账号的增删改查（系统管理员专属）

**角色**：系统管理员、教务管理人员、专业负责人、课程主讲教师

**核心 API**：
- `POST /api/auth/login` — 登录认证，返回 JWT Token + 用户基本信息
- `GET /api/auth/me` — 获取当前用户信息（用于前端路由守卫与 Pinia Store 初始化）
- `GET /api/admin/users` — 查询用户列表（admin 专属）
- `POST /api/admin/users` — 创建用户（admin 专属）
- `PATCH /api/admin/users/{user_id}` — 编辑用户信息或启用/禁用账号（admin 专属）

**权限模型**：RBAC，通过 FastAPI Depends 在路由级鉴权，校验 token 持有者 role 是否匹配接口要求。

**变更说明**：用户管理接口已从 `/api/auth/users` 迁移至 `/api/admin/users`，语义更清晰。

### 2.2 全局配置管理模块 (Admin Service)

**对应子模块**：M1-1 全局数据字典

**功能**：
- 系统级配置项管理：学年、学期、学院、专业等枚举值
- 支持按类别（category）筛选查询
- 增/删/改具体条目，key 唯一性校验

**核心 API**：
- `GET /api/admin/config?category=academic_year` — 按类别查询配置项列表
- `POST /api/admin/config` — 新增配置项
- `PUT /api/admin/config/{config_id}` — 更新配置项
- `DELETE /api/admin/config/{config_id}` — 删除配置项

**权限**：系统管理员（admin）

### 2.3 教务管理模块 (Academic Service)

**对应子模块**：M1-4 课程体系管理、教学班管理、学生名单 Excel 导入、M4-4 计算进度看板、M4-5 专业级全局计算（与 director 共享）、M5-2 专业级报告导出、成绩解锁

**功能**：
- 专业培养方案课程库的 CRUD（课程代码、名称、学分、学时、所属学院/专业、开课学年/学期）及批量导入
- 教学班创建与查询（关联课程、主讲教师、学年、学期）
- 学生名单 Excel 批量导入（模板格式：学号 + 姓名列），采用 Upsert 模式，返回插入/更新/错误明细
- 全专业计算进度看板（locked/pending 状态展示，含班级维度）
- 专业级全局计算触发（Tier 3），含前置校验（所有支撑课程均已锁定）
- 专业级报告导出（雷达图 + 穿透式台账 Excel）
- 成绩解锁管理（工单日志记录）

**核心 API**：
- `POST /api/academic/courses` — 新增课程
- `POST /api/academic/courses/import` — Excel 批量导入课程
- `GET /api/academic/courses` — 查询课程列表
- `DELETE /api/academic/courses/{course_id}` — 删除课程（前置依赖检查，存在关联教学班或矩阵记录则返回 409）
- `POST /api/academic/classes` — 创建教学班（含 teacher_id）
- `GET /api/academic/classes?course_id={course_id}` — 查询某课程下的班级列表
- `POST /api/academic/students/import?class_id={class_id}` — Excel 批量导入学生名单（Upsert）
- `GET /api/academic/students?class_id={class_id}` — 查询班级学生列表
- `GET /api/academic/calculation/progress` — 计算进度看板
- `POST /api/academic/calculate/program` — 专业级全局计算（Tier 3），权限：director/academic/admin
- `POST /api/academic/unlock` — 成绩解锁（academic/admin）
- `GET /api/academic/results/program` — 专业级报告数据（用于雷达图）
- `GET /api/academic/reports/program-excel` — 专业级穿透式 Excel 台账导出

**权限**：教务管理人员（academic）、专业负责人（director，部分接口共享）

**变更说明**：
- 新增课程批量导入、成绩解锁、专业报告导出独立接口
- 学生导入改为 Upsert 并返回详细统计
- 删除操作增加依赖检查，防止误删
- 全局计算接口路径调整至 `/api/academic/calculate/program`，并增加前置校验逻辑

### 2.4 专业设计模块 (Director Service)

**对应子模块**：M1-2 毕业要求管理、M1-3 指标点管理、M2-1/M2-2 宏观支撑矩阵配置与校验

**功能**：
- 毕业要求（一级）及指标点（二级）的结构化 CRUD
- 查询毕业要求时嵌套返回其下所有指标点
- 宏观支撑矩阵（课程 × 指标点）的配置与校验，采用全量替换提交
- 权重列合计严格校验：每个指标点下所有课程支撑权重之和 = 1.0（允许 ±0.001 误差）
- 前端实时显示列合计，不满足要求时标红并禁止提交

**核心 API**：
- `POST /api/director/requirements` — 新增毕业要求
- `GET /api/director/requirements` — 查询毕业要求列表（含嵌套指标点，SQLAlchemy selectinload 预加载）
- `DELETE /api/director/requirements/{req_id}` — 删除毕业要求（存在关联指标点时返回 409）
- `POST /api/director/indicators` — 新增指标点
- `GET /api/director/indicators?requirement_id={req_id}` — 按毕业要求筛选指标点
- `DELETE /api/director/indicators/{indicator_id}` — 删除指标点（存在关联矩阵记录时返回 409）
- `GET /api/director/macro-matrix` — 查询宏观矩阵已有数据
- `POST /api/director/macro-matrix` — 提交矩阵配置（全量替换，后端列合计校验）

**权限**：专业负责人（director）、系统管理员（admin）

**数据校验核心**：
- 提交矩阵时，后端按 indicator_id 分组计算 ΣW
- 存在 abs(ΣW - 1.0) > 0.001 → 返回 400，错误信息指明具体指标点
- 全部通过 → 事务（REPEATABLE READ）内 DELETE 全部旧矩阵数据 + INSERT 新数据（仅权重 > 0 的记录）

**变更说明**：矩阵提交策略改为全量替换，杜绝脏数据残留；前端增加实时校验与提交拦截。

### 2.5 课程主讲教师模块 (Teacher Service)

**对应子模块**：M3-1 课程目标设定、M3-2 内部权重分配、M3-3 考核点配置、M4-1 Excel 成绩模板导出、M4-2 成绩导入与预览、M4-3 课程级一键计算、M5-1 课程级评价报表导出

**功能**：
- 课程目标 CRUD（目标编号、描述、维度：知识/能力/价值）
- 内部权重矩阵（课程目标 × 指标点）配置与校验
- 考核点配置（考核点名称、满分分值、绑定课程目标）
- 复合表头 Excel 成绩模板流式下载
- 成绩 Excel 上传解析与 Web 预览，导入时严格校验考核点表头完全匹配
- 成绩聚合查询接口，返回"学生 × 考核点"二维矩阵
- 课程级一键计算（Tier 1 + Tier 2），计算完成后自动锁定（包括 objective-level 结果）
- 课程级《课程目标达成情况评价表》Excel 导出

**核心 API**：
- `GET /api/teacher/objectives?course_id={course_id}` — 查询课程全部目标
- `POST /api/teacher/objectives` — 新增课程目标（归属校验：仅限本课程教学班教师）
- `DELETE /api/teacher/objectives/{objective_id}` — 删除课程目标（存在依赖考核点或内部权重时返回 409）
- `POST /api/teacher/internal-weights?course_id={course_id}` — 提交内部权重矩阵
- `GET /api/teacher/internal-weights?course_id={course_id}` — 查询内部权重
- `GET /api/teacher/assessment-items?course_id={course_id}` — 查询全部考核点
- `POST /api/teacher/assessment-items` — 新增考核点
- `DELETE /api/teacher/assessment-items/{item_id}` — 删除考核点（存在学生成绩时返回 409）
- `GET /api/teacher/assessment-items/{course_id}/export-template` — 导出成绩模板（StreamingResponse）
- `POST /api/teacher/scores/import?course_id={course_id}&class_id={class_id}` — 导入成绩 Excel（含表头校验及归属校验）
- `GET /api/teacher/scores/matrix?course_id={course_id}&class_id={class_id}` — 成绩矩阵聚合查询
- `POST /api/teacher/calculate?course_id={course_id}&class_id={class_id}` — 课程级一键计算（Tier 1 + Tier 2）
- `GET /api/teacher/results?course_id={course_id}&class_id={class_id}` — 课程级报告导出

**权限**：课程主讲教师（teacher）、系统管理员（admin）

**业务规则**：
- 导入成绩前校验课程是否已锁定（已锁定则返回 400）
- 内部权重矩阵按 indicator_id 分组校验 Σw = 1.0
- 计算完成后自动锁定课程状态（level='course' AND locked=true），同时 objective-level 结果也设为 locked=true
- 教师只能操作其所属教学班的课程数据（基于 class.teacher_id）

**变更说明**：新增成绩矩阵接口，替代原来的多次请求组合；教师操作全面增加课程归属校验；考核点、课程目标删除增加依赖检查；课程级计算锁定范围扩展至 objective-level。

### 2.6 计算引擎 (Calculation Engine)

**对应子模块**：M4-3 课程级一键计算、M4-5 专业级全局计算

**核心流程**：

**Tier 1 — 课程目标级计算（C̄j）：**
1. 查该课程全部 assessment_items（含 objective_id, max_score）
2. 查该班级全部 students
3. 查 student_scores（关联 items 和 students）
4. Pandas DataFrame 分组聚合：
   - `df.groupby(['student_id','objective_id']).agg(score_sum, max_sum)`
   - `Cij = score_sum / max_sum`
   - `C̄j = Cij.mean()` per objective

**Tier 2 — 课程级计算（Ek）：**
1. 查 internal_weight_matrix（通过 objectives 关联）
2. 对每个 indicator：`Ek = Σ (C̄j × weight_w)`

**Tier 3 — 专业级计算（Gk）：**
1. 前置校验：对每个指标点，确认其全部支撑课程（macro_support_matrix）均已拥有 level='course' AND locked=true 的结果，否则返回 400 并指明缺失课程
2. 查所有 level='course' AND locked=true 的 calculation_results
3. 查 macro_support_matrix 全部记录
4. 对每个 indicator：`Gk = Σ (Ek × W)`，对所有支撑该 indicator 的课程求和

**状态锁定机制**：
- Tier 1 + Tier 2 完成后，INSERT level='objective' AND locked=true 及 level='course' AND locked=true 结果记录
- 已锁定课程拒绝成绩导入请求
- 所有课程 locked 后，Tier 3 「执行全局计算」按钮可用

**数据持久化**：
- 每次计算前 DELETE 旧结果（同一 course_id + class_id 范围），再 INSERT 新结果
- 事务隔离级别 REPEATABLE READ，保证原子性与一致性

**变更说明**：objective-level 结果同样锁定；全局计算增加前置校验；明确事务隔离级别。

## 3. 前端空间与核心页面设计

系统前端采用 Vue 3 单页应用（SPA）架构，结合 Element Plus 组件库，划分为 5 个核心工作空间，共计 11 个核心业务页面：

### 3.1 接入与公共空间 (Access Space)

**P-1 Horizon Portal（登录门户 — Login.vue）**
- 权限：全部用户
- 描述：系统唯一认证入口。用户输入用户名和密码，调用 `POST /api/auth/login`，登录成功后 JWT 存储至 localStorage，role 存入 Pinia Auth Store，路由跳转至对应角色的 Dashboard。

---

### 3.2 教务治理空间 (Governance Space)

**P-2 MetaDictionary（元数据中心 — DictConfig.vue）**
- 权限：系统管理员
- 描述：统管全校级基础字典。调用 `/api/admin/config` 系列接口，对学年、学期、学院、专业等枚举值进行 CRUD 管理。
- 操作：选择配置类别 → 查询现有条目 → 增/删/改条目。

**P-3 NexusRoster（学籍与课表网关 — CourseStudentImport.vue）**
- 权限：教务管理人员
- 描述：全专业培养方案课程库及学生学籍名单的初始化中心。
- 操作：
  - 课程管理：调用 `/api/academic/courses` 系列接口（含 `POST /api/academic/courses/import` 批量导入）
  - 教学班管理：调用 `/api/academic/classes` 系列接口（创建时指定主讲教师）
  - 学生导入：选择目标班级 → 上传 Excel 文件 → 调用 `POST /api/academic/students/import?class_id={class_id}`，界面展示导入统计（插入、更新、错误详情）

**P-4 PulseMonitor（全景算力雷达 — ProgressMonitor.vue）**
- 权限：教务管理人员 / 专业负责人
- 描述：实时进度监控看板。调用 `GET /api/academic/calculation/progress`，以表格形式展示全专业所有课程班级的锁定状态：
  - locked → 绿色「已锁定」标签
  - pending → 橙色「待计算」标签
- 核心交互：所有课程班级 locked 时，下方「执行全局计算」按钮高亮可用，点击触发 `POST /api/academic/calculate/program`。

---

### 3.3 专业设计空间 (Blueprint Space)

**P-5 OutcomeBlueprint（毕业要求设计器 — GraduationRequirements.vue）**
- 权限：专业负责人
- 描述：毕业要求（一级）与指标点（二级）的结构化编辑器。
- 操作：
  - 查询：调用 `GET /api/director/requirements`（返回嵌套指标点的树形结构）
  - 新增毕业要求：调用 `POST /api/director/requirements`
  - 新增指标点：下拉选择父级毕业要求，调用 `POST /api/director/indicators`
  - 删除：存在依赖时前端提示 409 错误

**P-6 MacroMatrix Studio（宏观矩阵工坊 — MacroMatrix.vue）**
- 权限：专业负责人
- 描述：高密度宏观权重矩阵分配工作台。
- 初始加载（3 次独立查询）：
  - `GET /api/academic/courses` → 行数据（全部课程）
  - `GET /api/director/indicators` → 列数据（全部指标点）
  - `GET /api/director/macro-matrix` → 已有矩阵数据预填
- 核心交互：
  - 十字高亮 DataGrid，纵轴为课程、横轴为指标点
  - 勾选支撑关系，输入权重（小数，0.00~1.00）
  - 前端实时校验：每个指标点列底部显示当前 ΣW，偏离 1.0 时该列标红，提交按钮保持禁用
  - 全部通过后点击「提交」，调用 `POST /api/director/macro-matrix`（全量数据）
  - 后端再次校验，不通过则返回具体错误信息

---

### 3.4 课程评价空间 (Evaluation Space)

**P-7 MyCourse Hub（我的教学空间 — MyCourses.vue）**
- 权限：课程主讲教师
- 描述：教师个人仪表盘。展示本学期任教的课程教学班卡片，包含当前课程的计算阶段指示器及锁定状态。

**P-8 MicroSyllabus（大纲微观映射器 — SyllabusConfig.vue）**
- 权限：课程主讲教师
- 描述：教学大纲量化映射器。包含课程目标设定和内部权重分配两个子功能。
- 操作：
  - 课程目标 CRUD：调用 `/api/teacher/objectives` 系列接口
  - 内部权重矩阵：前端加载本课程目标 + 宏观矩阵中本课程支撑的指标点，构建「课程目标 × 指标点」网格
  - 提交内部权重：调用 `POST /api/teacher/internal-weights?course_id={course_id}`
  - 后端按 indicator_id 分组校验 Σw = 1.0

**P-9 Assessment Mapper（考核点追溯板 — AssessmentMapping.vue）**
- 权限：课程主讲教师
- 描述：考核点绑定配置页。
- 操作：
  - 查询：`GET /api/teacher/assessment-items?course_id={course_id}`
  - 新增：输入考核点名称、满分分值，下拉选择绑定课程目标，调用 `POST /api/teacher/assessment-items`
  - 删除：若已有成绩数据，后端返回 409 并提示

**P-10 GradeForge（动态成绩工场 — GradeEntry.vue）**
- 权限：课程主讲教师
- 描述：数据录入与一键点火计算核心页。
- 操作流程：
  1. 导出模板：调用 `GET /api/teacher/assessment-items/{course_id}/export-template`，浏览器自动下载 Excel（复合表头：Row 1 考核点名称，Row 2 满分分值 + 绑定目标）
  2. 导入成绩：选择目标班级，上传填写好的 Excel，调用 `POST /api/teacher/scores/import?course_id={course_id}&class_id={class_id}`。系统强制校验表头考核点完全匹配，不一致则拒绝并提示差异。界面展示导入统计。
  3. Web 预览：调用 `GET /api/teacher/scores/matrix?course_id={course_id}&class_id={class_id}` 获取"学生×考核点"矩阵，直接渲染 DataGrid，空单元格高亮提示
  4. 一键计算：点击「计算课程达成度」，调用 `POST /api/teacher/calculate?course_id={course_id}&class_id={class_id}`
  5. 结果显示：展示 Tier 1（课程目标 C̄j）和 Tier 2（指标点 Ek）结果表格
  6. 计算完成后成绩自动锁定，导入按钮禁用，若需勘误需联系教务管理员解锁

### 4.2 数据校验规范

| 校验项 | 校验规则 | HTTP 状态码 |
|------|------|:---:|
| JWT Token 有效性 | 解析成功 + 用户存在 + is_active=true | 401 |
| RBAC 角色权限 | Token role ∈ 接口允许角色集合 | 403 |
| 教师归属校验 | 当前用户为课程关联教学班的 teacher_id | 403 |
| 用户名唯一性 | SELECT COUNT(*) FROM users WHERE username=? | 400 |
| 配置项 key 唯一性 | SELECT COUNT(*) FROM global_config WHERE key=? AND category=? | 400 |
| 课程代码唯一性 | SELECT COUNT(*) FROM courses WHERE code=? | 400 |
| 毕业要求编号唯一性 | SELECT COUNT(*) FROM graduation_requirements WHERE code=? | 400 |
| 宏观矩阵列合计 | abs(ΣW−1.0) ≤ 0.001 | 400 |
| 内部权重列合计 | abs(Σw−1.0) ≤ 0.001 | 400 |
| 成绩导入锁定检查 | SELECT locked FROM calculation_results WHERE ... | 400 |
| 成绩导入考核点表头校验 | 上传文件的考核点集合必须与系统定义完全一致 | 400 |
| 全局计算前置校验 | 每个指标点的全部支撑课程均已锁定 | 400 |
| Excel 列名校验 | 学生导入须含"学号""姓名"；成绩模板须含相应考核点 | 400 |
| 删除依赖检查 | 存在关联数据时返回 409，提示需先清理的依赖 | 409 |

## 6. 数据流向

1. 基础数据配置
   - 管理员配置全局字典（学年/学期/学院/专业）
   - ↓
   - 专业负责人设计毕业要求及指标点
   - ↓
   - 教务管理员导入课程库（支持批量 Excel）与教学班学生名单（Upsert）

2. 矩阵配置
   - 专业负责人配置宏观支撑矩阵（课程×指标点，W，全量替换式提交）
   - ↓
   - 教师设定课程目标
   - ↓
   - 教师配置内部权重矩阵（课程目标×指标点，w）
   - ↓
   - 教师配置考核点（绑定课程目标）

3. 成绩处理
   - 教师导出成绩模板（复合表头 Excel）
   - ↓
   - 教师分发模板、收集成绩、上传导入（严格表头匹配）
   - ↓
   - Web 端成绩矩阵预览（一体化聚合接口）

4. 计算引擎
   - 教师触发课程级计算（Tier 1 + Tier 2）
   - ├─ Tier 1: 考核点得分 → 课程目标达成度 C̄j
   - └─ Tier 2: C̄j × 内部权重 w → 课程级指标点达成度 Ek
   - ↓
   - 课程全部结果锁定（objective + course 级）
   - ↓
   - 教务/专业负责人触发专业级计算（Tier 3，前置校验所有课程已锁定）
   - └─ Tier 3: Ek × 宏观权重 W → 专业级指标点达成度 Gk

5. 报告导出
   - 教师导出课程级《课程目标达成情况评价表》
   - ↓
   - 专业负责人导出专业级雷达图（轴数=指标点数，参考圆环）+ 穿透式台账 Excel

6. 勘误流程
   - 教师在已锁定状态需修改成绩 → 向教务管理员申请
   - 教务管理员调用 `POST /api/academic/unlock`，填写理由
   - 解锁后教师可重新导入成绩并计算

## 8. 安全设计

- **认证**：JWT Bearer Token（有效期 8 小时），结合 Redis 实现 Token 黑名单与单点登录挤出（后续迭代）
- **授权**：RBAC 模型，以 FastAPI Depends 依赖注入方式在路由级鉴权
- **资源归属校验**：教师只能操作自己担任主讲的教学班数据，通过 class.teacher_id 与 token 中的 user_id 比对
- **数据安全**：
  - 计算快照留存（calculation_results 表），锁定后禁止直接修改
  - 不可逆哈希密码存储（bcrypt）
  - 严格的成绩状态机隔离（一旦 locked=true，拒绝成绩 C/U/D 请求）
  - 成绩解锁操作记录日志（unlock_log）
- **API 安全**：
  - CORS 跨域白名单
  - XSS 防御（Pydantic 自动转义）
  - API 限流保护（Rate Limiter，后续迭代）
- **数据校验**：Pydantic V2 严格模式，浮点运算容差 1e-3
- **事务安全**：计算、矩阵提交、成绩导入等关键操作均包裹在数据库事务中，隔离级别 REPEATABLE READ，异常自动回滚
- **删除保护**：对存在依赖数据的资源执行删除操作时，返回 409 Conflict，防止误删
