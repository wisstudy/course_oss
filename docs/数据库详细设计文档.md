## 数据库详细设计文档

### 1. 数据库概述

- **数据库名称**：achievement_calc
- **数据库引擎**：MySQL 8.0+ (InnoDB)
- **默认事务隔离级别**：REPEATABLE READ
- **字符集**：utf8mb4，排序规则 utf8mb4_unicode_ci
- **主键策略**：统一使用 UUID v4，存储为 CHAR(36)
- **时间戳**：所有业务表均包含 created_at 和 updated_at 字段，用于记录数据创建和最后修改时间
- **软删除**：不做物理删除的数据表（如用户、课程等）使用 is_active 或 is_deleted 标记；关键业务数据删除前由应用层检查外键依赖，存在依赖时拒绝删除并返回 409

##### 2.3.3 macro_support_matrix 宏观支撑矩阵

| 字段名 | 数据类型 | 约束 | 默认值 | 注释 |
|--------|---------|------|--------|------|
| id | CHAR(36) | PRIMARY KEY | UUID | |
| course_id | CHAR(36) | NOT NULL, FOREIGN KEY | | 课程 |
| indicator_id | CHAR(36) | NOT NULL, FOREIGN KEY | | 指标点 |
| weight_W | DECIMAL(5,4) | NOT NULL | | 宏观支撑权重 W，范围 [0,1] |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | |

**索引**:
- UNIQUE idx_course_indicator (course_id, indicator_id) — 同一课程对同一指标点只能有一个权重
- INDEX idx_indicator (indicator_id)

**外键**:
- FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
- FOREIGN KEY (indicator_id) REFERENCES graduation_indicators(id) ON DELETE CASCADE

**说明**:
- 矩阵提交采用全量替换策略：事务中先 `DELETE FROM macro_support_matrix` 清除所有旧记录，再批量 INSERT 新配置。
- 应用层负责在提交前按指标点维度校验 ΣW=1.0（容差 0.001），不通过则返回 400 错误。

#### 2.5 计算结果表

##### 2.5.1 calculation_results 计算结果快照表

| 字段名 | 数据类型 | 约束 | 默认值 | 注释 |
|--------|---------|------|--------|------|
| id | CHAR(36) | PRIMARY KEY | UUID | |
| course_id | CHAR(36) | NOT NULL, FOREIGN KEY | | 关联课程（专业级计算时可为 NULL 或关联虚拟课程） |
| class_id | CHAR(36) | NULL, FOREIGN KEY | | 关联教学班（课程级计算必填，专业级为空） |
| level | ENUM('objective','course','program') | NOT NULL | | 计算级别：目标级 / 课程级 / 专业级 |
| objective_id | CHAR(36) | NULL | | 当 level='objective' 时，关联的课程目标 |
| indicator_id | CHAR(36) | NULL | | 当 level='course' 或 'program' 时，关联的指标点 |
| achievement_value | DECIMAL(6,4) | NOT NULL | | 达成度数值 |
| locked | TINYINT(1) | NOT NULL | 0 | 是否锁定（课程级计算后，objective 和 course 级均锁定） |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE | |

**索引**:
- INDEX idx_course_class_level (course_id, class_id, level) — 查询某课程某班级所有计算结果
- INDEX idx_level_locked (level, locked) — 快速查找已锁定的课程级/专业级结果（用于全局计算前置校验）
- INDEX idx_indicator (indicator_id)

**外键**:
- FOREIGN KEY (course_id) REFERENCES courses(id) ON DELETE CASCADE
- FOREIGN KEY (class_id) REFERENCES classes(id) ON DELETE CASCADE
- FOREIGN KEY (objective_id) REFERENCES course_objectives(id) ON DELETE SET NULL
- FOREIGN KEY (indicator_id) REFERENCES graduation_indicators(id) ON DELETE SET NULL

**修订说明**:
- 锁定范围扩展：课程级计算完成后，不仅 course 级结果锁定（locked=1），objective 级结果也必须锁定，以保证数据一致性。解锁操作会同时将两种级别的 locked 设为 0。
- 专业级计算前，需通过前置校验：利用 macro_support_matrix 与 calculation_results 确认每一个指标点的全部支撑课程均存在 locked=1 的 course 级记录。

### 3. 关系图与关联规则

- users 与 classes 通过 teacher_id 关联，一个教师可讲授多个教学班。
- courses 与 classes 为一对多，一个课程可有多个教学班。
- students 与 class_students 为一对多，一个学生可选修多个教学班；classes 与 class_students 为一对多，形成多对多选课关系。
- student_scores 的 class_student_id 直接引用 class_students.id，将成绩绑定到具体的选课记录，彻底消除学生多班级时的关联歧义。
- graduation_requirements 与 graduation_indicators 为一对多，级联删除。
- macro_support_matrix 连接 courses 和 graduation_indicators，形成多对多关系，并携带权重；全量替换式提交。
- course_objectives 属于一个 courses。
- internal_weight_matrix 连接 course_objectives 和 graduation_indicators，形成多对多关系。
- assessment_items 属于一个 courses，并绑定到一个 course_objectives。
- calculation_results 存储各级计算结果，可追溯到课程、班级、目标、指标点；锁定策略覆盖 objective 和 course 两级。
- unlock_log 记录成绩解锁操作，关联课程、班级与操作人。

### 4. 索引策略总结

| 表名 | 索引 | 用途 |
|------|------|------|
| users | UNIQUE(username) | 登录唯一性检查 |
| global_config | UNIQUE(category, key) | 字典项唯一性 |
| courses | UNIQUE(code) | 课程代码唯一性 |
| classes | INDEX(course_id), INDEX(teacher_id) | 按课程或教师查询班级 |
| students | UNIQUE(student_id) | 学号全局唯一，支持 Upsert |
| class_students | UNIQUE(class_id, student_id), INDEX(student_id) | 选课关系唯一约束，按学生查所有课程 |
| graduation_requirements | UNIQUE(code) | 编号唯一 |
| graduation_indicators | UNIQUE(code) | 指标点编号全局唯一 |
| macro_support_matrix | UNIQUE(course_id, indicator_id) | 一个课程对同一指标点只有一个权重；INDICATOR 索引加速按指标点求和 |
| course_objectives | UNIQUE(course_id, code) | 课程内目标编号唯一 |
| internal_weight_matrix | UNIQUE(objective_id, indicator_id) | 一个目标对同一指标点只有一个权重 |
| assessment_items | INDEX(course_id) | 按课程快速检索考核点 |
| student_scores | UNIQUE(class_student_id, assessment_item_id) | 每个选课记录每个考核点一条成绩，支持 Upsert |
| calculation_results | INDEX(course_id, class_id, level), INDEX(level, locked), INDEX(indicator_id) | 多维度检索计算结果，全局计算前置校验依赖 locked 索引 |
| unlock_log | INDEX(course_id, class_id) | 按课程班级查询解锁历史 |

### 5. 关键业务规则与实现说明

- **删除保护**：对存在下游依赖记录的删除操作（如删除已有班级的课程、已有成绩的考核点等），应用层会先查询关联表，存在则返回 HTTP 409 Conflict，提示需要先清理哪些依赖，禁止直接物理删除。

- **学生名单导入（Excel）**：
  1. 解析学号、姓名、班级 ID 等信息。
  2. 在 students 表中按学号执行 `INSERT ... ON DUPLICATE KEY UPDATE name=VALUES(name)`，确保学生基本信息存在且姓名可更新。
  3. 根据学号查询 students.id，然后在 class_students 表中按 (class_id, student_id) 执行 Upsert（或 INSERT IGNORE），建立选课关系。
  4. 返回导入统计（inserted/updated/errors）。

- **成绩导入**：
  - 导入时需包含学号与班级 ID（或直接提供 class_student_id）。
  - 若仅有学号和班级 ID，先查询 class_students 获取选课记录 ID。
  - 基于 (class_student_id, assessment_item_id) 执行 `INSERT ... ON DUPLICATE KEY UPDATE score=VALUES(score)`。

- **成绩锁定**：教师完成课程级计算后，calculation_results 中该班级的 objective 和 course 级别记录 locked 均置为 1。此后成绩导入接口检查锁定状态，已锁定则返回 400。

- **解锁流程**：教务管理员调用解锁接口，系统将相关 locked 设为 0，并向 unlock_log 插入审计记录。

- **矩阵全量替换**：宏观支撑矩阵提交时，后端在事务中先 DELETE 全表，再 INSERT 新数据，保证配置的原子性和一致性。

- **全局计算前置校验**：专业级计算前，系统会按指标点逐项检查其所有支撑课程（macro_support_matrix）是否均存在 `level='course' AND locked=1` 的计算结果，若缺失则返回 400 并指明未完成课程列表。

- **事务隔离**：所有计算、矩阵提交、成绩锁定等核心写操作均在 REPEATABLE READ 隔离级别下执行，防止并发导致的幻读或计算不一致。
