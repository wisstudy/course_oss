### 2.4 专业设计模块 (Director Service)

**对应子模块**：M1-2 毕业要求管理、M1-3 指标点管理、M2-1/M2-2 宏观支撑矩阵配置与校验

**功能**：
- 毕业要求（一级）及指标点（二级）的结构化 CRUD
- 查询毕业要求时嵌套返回其下所有指标点
- 宏观支撑矩阵（课程 × 指标点）的配置与校验
- 权重列合计严格校验：每个指标点下所有课程支撑权重之和 = 1.0（允许 ±0.001 误差）

**核心 API**：
- `POST /api/director/requirements` — 新增毕业要求
- `GET /api/director/requirements` — 查询毕业要求列表（含嵌套指标点，SQLAlchemy selectinload 预加载）
- `DELETE /api/director/requirements/{req_id}` — 删除毕业要求（级联删除指标点及矩阵关联数据）
- `POST /api/director/indicators` — 新增指标点
- `GET /api/director/indicators?requirement_id={req_id}` — 按毕业要求筛选指标点
- `DELETE /api/director/indicators/{indicator_id}` — 删除指标点
- `GET /api/director/macro-matrix` — 查询宏观矩阵已有数据
- `POST /api/director/macro-matrix` — 提交矩阵配置（含列合计校验）

**权限**：专业负责人（director）、系统管理员（admin）

**数据校验核心**：
- 提交矩阵时，后端按 indicator_id 分组计算 ΣW
- 存在 abs(ΣW - 1.0) > 0.001 → 返回 400，错误信息指明具体指标点
- 全部通过 → 事务内 DELETE 旧数据 + INSERT 新数据

### 2.6 计算引擎 (Calculation Engine)

**对应子模块**：M4-3 课程级一键计算、M4-5 专业级全局计算

**核心流程**：

**Tier 1 — 课程目标级计算（C̄j）：**
1. 查该课程全部 assessment_items（含 objective_id, max_score）
2. 查该班级全部 students
3. 查 student_scores（关联 items 和 students）
4. Pandas DataFrame 分组聚合：
   - `df.groupby(['student_id','objective_id']).agg(score_sum, max_sum)`
   - `Cij = score_sum / max_sum`
   - `C̄j = Cij.mean()` per objective

**Tier 2 — 课程级计算（Ek）：**
1. 查 internal_weight_matrix（通过 objectives 关联）
2. 对每个 indicator：`Ek = Σ (C̄j × weight_w)`

**Tier 3 — 专业级计算（Gk）：**
1. 查所有 `level='course' AND locked=true` 的 calculation_results
2. 查 macro_support_matrix 全部记录
3. 对每个 indicator：`Gk = Σ (Ek × W)`，对所有支撑该 indicator 的课程求和

**状态锁定机制**：
- Tier 1 + Tier 2 完成后，INSERT `level='course' AND locked=true` 结果记录
- 已锁定课程拒绝成绩导入请求
- 所有课程 locked 后，Tier 3 「执行全局计算」按钮可用

**数据持久化**：
- 每次计算前 DELETE 旧结果（同一 course_id + class_id 范围），再 INSERT 新结果
- 事务保证原子性

### 4.2 核心 API 详细设计

#### 4.2.1 登录认证 API

**POST /api/auth/login**

请求体：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

后端操作：

1. 查 users 表，条件：username = ? AND is_active = true
2. 用户不存在或已禁用 → 401
3. bcrypt.verify(明文密码, password_hash) 验证密码
4. 不匹配 → 401
5. 匹配 → 用 user.id 和 user.role 生成 JWT（过期时间 8 小时）
6. 返回 token + 用户基本信息

响应体：

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer",
  "role": "admin",
  "display_name": "系统管理员"
}
```

**GET /api/auth/me**

请求头：`Authorization: Bearer <token>`

后端操作：解析 JWT → 查 users 表 → 返回用户信息（id, username, role, display_name, college, major, is_active）

#### 4.2.2 宏观支撑矩阵 API

**POST /api/director/macro-matrix**

请求体：

```json
[
  {
    "course_id": "course-uuid-ds",
    "indicator_id": "ind-uuid-3-1",
    "weight_W": 0.3
  },
  {
    "course_id": "course-uuid-ds",
    "indicator_id": "ind-uuid-3-3",
    "weight_W": 0.5
  }
]
```

后端操作：

1. RBAC 校验 role ∈ {director, admin}
2. 按 indicator_id 分组，计算每个 indicator 的 ΣW
3. 存在 abs(ΣW - 1.0) > 0.001 → 返回 400，错误信息指明具体指标点
4. 全部通过 → 事务开始
5. DELETE FROM macro_support_matrix WHERE course_id IN (...)（删除涉及课程的旧数据）
6. INSERT INTO macro_support_matrix (...) 批量插入
7. 提交事务，返回全量矩阵数据

#### 4.2.3 课程级计算 API

**POST /api/teacher/calculate?course_id={course_id}&class_id={class_id}**

后端操作：

1. RBAC 校验 role ∈ {teacher, admin}
2. 检查 locked → 已锁定则 400
3. Tier 1 计算：
   - 查全部 assessment_items + students + student_scores
   - Pandas DataFrame 分组聚合计算 C̄j
4. Tier 2 计算：
   - 查 internal_weight_matrix
   - 对每个 indicator：Ek = Σ (C̄j × weight_w)
5. 持久化：
   - DELETE 旧 calculation_results（同 course_id + class_id）
   - INSERT objective-level 结果（locked=false）
   - INSERT course-level 结果（locked=true）
6. 返回计算结果

响应体：

```json
{
  "objective_results": [
    { "objective_id": "obj-uuid-1-1", "C_bar": 0.8750 },
    { "objective_id": "obj-uuid-2-1", "C_bar": 0.7600 }
  ],
  "course_results": {
    "ind-uuid-3-1": 0.8750,
    "ind-uuid-3-3": 0.7915
  }
}
```

#### 4.2.4 专业级计算 API

**POST /api/teacher/calculate/program**

后端操作：

1. RBAC 校验 role ∈ {director, academic, admin}
2. 查所有 level='course' AND locked=true 的 calculation_results → 得到每门课程的 Ek
3. 查 macro_support_matrix 全部记录 → 得到每个 (course, indicator) 的 W
4. 对每个 indicator：Gk = Σ (Ek × W)
5. DELETE 旧 program-level 结果
6. INSERT INTO calculation_results (level='program', indicator_id, achievement_value=Gk, locked=true)
7. 返回结果

响应体：

```json
{
  "program_results": {
    "indicator-uuid-3-1": 0.8520,
    "indicator-uuid-3-3": 0.7915,
    "indicator-uuid-3-4": 0.9150
  }
}
```

#### 4.2.6 成绩导入 API

**POST /api/teacher/scores/import?course_id={course_id}&class_id={class_id}**

请求：multipart/form-data，文件字段 file

后端操作：

1. RBAC 校验 role ∈ {teacher, admin}
2. 查 locked 状态 → 已锁定则 400 "该课程成绩已锁定，请联系教务管理员解锁"
3. pandas 解析 Excel（header=None）
4. Row 0 = 表头解析考核点名称
5. Row 2+ 逐行处理：
   - 取 student_id → 查 students 表匹配 → 得到 student.id
   - 逐列匹配考核点名称 → 得到 assessment_item_id
   - Upsert 到 student_scores
6. 返回导入计数：`{ "count": 25 }`

### 4.3 数据校验规范

| 校验项 | 校验规则 | HTTP 状态码 |
|------|------|:---:|
| JWT Token 有效性 | 解析成功 + 用户存在 + is_active=true | 401 |
| RBAC 角色权限 | Token role ∈ 接口允许角色集合 | 403 |
| 用户名唯一性 | SELECT COUNT(*) FROM users WHERE username=? | 400 |
| 配置项 key 唯一性 | SELECT COUNT(*) FROM global_config WHERE key=? AND category=? | 400 |
| 课程代码唯一性 | SELECT COUNT(*) FROM courses WHERE code=? | 400 |
| 毕业要求编号唯一性 | SELECT COUNT(*) FROM graduation_requirements WHERE code=? | 400 |
| 宏观矩阵列合计 | \|ΣW−1.0\| ≤ 0.001 | 400 |
| 内部权重列合计 | \|Σw−1.0\| ≤ 0.001 | 400 |
| 成绩导入锁定检查 | SELECT locked FROM calculation_results WHERE ... | 400 |
| 计算重复触发检查 | 同成绩导入锁定检查 | 400 |
| Excel 列名校验 | 必须包含"学号"和"姓名"（或考核点名称） | 400 |
| 资源归属校验 | teacher 只能操作自己教学的课程/班级 | 403 |

## 6. 数据流向

1. 基础数据配置
   - 管理员配置全局字典（学年/学期/学院/专业）
   - ↓
   - 专业负责人设计毕业要求及指标点
   - ↓
   - 教务管理员导入课程库与教学班学生名单

2. 矩阵配置
   - 专业负责人配置宏观支撑矩阵（课程×指标点，W）
   - ↓
   - 教师设定课程目标
   - ↓
   - 教师配置内部权重矩阵（课程目标×指标点，w）
   - ↓
   - 教师配置考核点（绑定课程目标）

3. 成绩处理
   - 教师导出成绩模板（复合表头 Excel）
   - ↓
   - 教师分发模板、收集成绩、上传导入
   - ↓
   - Web 端成绩预览与微调

4. 计算引擎
   - 教师触发课程级计算（Tier 1 + Tier 2）
   - ├─ Tier 1: 考核点得分 → 课程目标达成度 C̄j
   - └─ Tier 2: C̄j × 内部权重 w → 课程级指标点达成度 Ek
   - ↓
   - 课程结果锁定
   - ↓
   - 教务/专业负责人触发专业级计算（Tier 3）
   - └─ Tier 3: Ek × 宏观权重 W → 专业级指标点达成度 Gk

5. 报告导出
   - 教师导出课程级《课程目标达成情况评价表》
   - ↓
   - 专业负责人导出专业级雷达图 + 穿透式台账

## 8. 安全设计

- 认证：JWT Bearer Token（有效期 8 小时），结合 Redis 实现 Token 黑名单与单点登录挤出（后续迭代）
- 授权：RBAC 模型，以 FastAPI Depends 依赖注入方式在路由级鉴权
- 资源归属校验：教师只能操作自己教学的课程/班级，通过 class_teachers 关联表校验
- 数据安全：
  - 计算快照留存（calculation_results 表）
  - 不可逆哈希密码存储（bcrypt）
  - 严格的成绩状态机隔离（一旦 locked=true，拒绝成绩 C/U/D 请求）
- API 安全：
  - CORS 跨域白名单
  - XSS 防御（Pydantic 自动转义）
  - API 限流保护（Rate Limiter，后续迭代）
- 数据校验：Pydantic V2 严格模式，浮点运算容差 1e-3
- 事务安全：计算与矩阵提交操作均包裹在数据库事务中，异常自动回滚
