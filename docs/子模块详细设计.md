## 角色一：系统管理员（4 个子模块）

### M0-1 登录认证

**功能说明**

用户输入用户名和密码，系统验证身份后签发 JWT Token，前端存储 Token 用于后续所有 API 请求的身份标识。

**用户输入**
- 用户名（文本）
- 密码（文本）

**前端 → 后端**

```
POST /api/auth/login
Content-Type: application/json
{
  "username": "admin",
  "password": "admin123"
}
```

**后端操作**

1. 查 users 表，条件：username = ? AND is_active = true
2. 用户不存在或已禁用 → 返回 401
3. bcrypt.verify(明文密码, password_hash) 验证密码
4. 不匹配 → 返回 401
5. 匹配 → 用 user.id 和 user.role 生成 JWT（过期时间 8 小时）
6. 返回 token + 用户基本信息

**后端 → 前端**

```json
{
  "access_token": "eyJhbGciOi...",
  "token_type": "bearer",
  "role": "admin",
  "display_name": "系统管理员"
}
```

**前端操作**
- 将 token 存入 localStorage
- 将 role 存入 Pinia auth store
- 路由跳转到对应角色的 Dashboard

### M4-5 专业级全局计算 (Tier 3)（修订：路径、前置校验）

**功能说明**

教务管理员/专业负责人在所有支撑课程 locked 后，点击按钮触发专业级达成度计算。

**用户输入**

点击"执行专业级计算"按钮

**前端 → 后端**

```
POST /api/academic/calculate/program
Authorization: Bearer <token>
```

**后端操作**

1. RBAC 校验 role ∈ {director, academic, admin}
2. 前置校验：
   - 获取所有指标点
   - 对每个指标点，从 macro_support_matrix 查询全部支撑课程
   - 检查每门支撑课程（所有对应班级）是否都有 level='course' AND locked=true 的计算结果
   - 若发现未锁定的课程，返回 400，并明确指出指标点及未完成课程列表：

```json
{
  "detail": "以下指标点的支撑课程尚未全部完成计算：指标点 3-3 缺少 [离散数学(2025级离散数学班)]"
}
```

3. 计算（事务隔离级别 REPEATABLE READ）：
   - 查所有 level='course' AND locked=true 的 calculation_results → 每门课程每个班级的 Ek
   - 查 macro_support_matrix → 每个 (course, indicator) 的 W
   - 对每个 indicator：Gk = Σ ( Ek × W )，对所有支撑该 indicator 的课程及其班级求平均后加权
4. 删除旧的 program-level 结果
5. INSERT INTO calculation_results (level='program', indicator_id, achievement_value=Gk, locked=true)
6. 提交事务，返回结果

**后端 → 前端**

```json
{
  "program_results": {
    "indicator-uuid-3-1": 0.8520,
    "indicator-uuid-3-3": 0.7915,
    "indicator-uuid-3-4": 0.9150
  }
}
```

**前端操作**

在页面展示 Gk 结果表格，每个指标点一行，显示达成度数值（四舍五入到 4 位小数）。

### M2-1 + M2-2 宏观支撑矩阵配置与校验（修订：全量替换 + 前端校验描述）

**功能说明**

专业负责人在二维网格中配置"课程→指标点"的支撑关系和总支撑权重 W，系统自动校验每列 ΣW = 1.0。

**用户输入**

- 进入矩阵配置页面（MatrixEditor 组件）
- 在网格中勾选支撑关系，输入权重（小数，0.00~1.00）
- 前端实时校验：每个指标点列底部实时显示当前权重和，若 abs(sum - 1.0) > 0.001 则该列标题及合计数字标红，提交按钮保持禁用状态；全部列通过后按钮启用。
- 点击"提交"时，前端组装所有课程的完整矩阵数据（包括未勾选的课程也作为空或权重0提交，或约定仅提交有权重的条目但需携带所有指标点维度）。

**初始加载数据（前端需要）**

```
GET /api/academic/courses        →  课程列表
GET /api/director/indicators     →  指标点列表
GET /api/director/macro-matrix   →  已有矩阵数据
```

**前端操作**

构建课程(行) × 指标点(列)网格，预填已有权重。

**提交矩阵（修订：全量替换）**

**前端 → 后端**

```
POST /api/director/macro-matrix
Authorization: Bearer <token>
{
  "matrix": [
    {
      "course_id": "course-uuid-ds",
      "indicator_id": "ind-uuid-3-1",
      "weight_W": 0.3
    },
    {
      "course_id": "course-uuid-ds",
      "indicator_id": "ind-uuid-3-3",
      "weight_W": 0.5
    }
    // ... 所有课程与指标点的组合，权重为 0 的可省略，后端自动补0
  ]
}
```

**后端操作**

1. RBAC 校验 role ∈ {director, admin}
2. 将前端传入的矩阵按 indicator_id 分组，计算 ΣW（缺失的组合视为 0）
3. 若任一指标点的 abs(ΣW - 1.0) > 0.001 → 返回 400，指明不合格指标点
4. 事务（REPEATABLE READ）：
   - DELETE FROM macro_support_matrix（删除全部旧数据）
   - INSERT INTO macro_support_matrix (...) 批量插入所有权重 > 0 的记录
5. 提交事务，返回全量矩阵数据

**后端 → 前端**

```json
{
  "matrix": [
    { "id": "new-uuid", "course_id": "...", "indicator_id": "...", "weight_W": 0.3 }
  ]
}
```

**前端操作**

矩阵刷新为最新保存结果，提示"保存成功"。

## 非功能补充

- **事务隔离级别**：所有涉及计算、锁定及矩阵替换的操作均在 REPEATABLE READ 隔离级别下执行，防止并发更新导致计算不一致。
- **前端矩阵校验**：宏观支撑矩阵配置页面中，每个指标点列底部实时显示 ΣW，若偏离 1.0 则标红并禁用提交按钮，直至全部通过。
- **雷达图规格**：专业级报告雷达图轴数等于指标点总数，刻度 0~1，参考线 0.25/0.5/0.75/1.0，连线显示数值标签。
